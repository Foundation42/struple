struple

Schema-free binary tuples whose bytes sort themselves.

Pack values into bytes that are directly memcmp-comparable — drop them into any byte-ordered store and they sort in the correct semantic order, no custom comparator. It's a stream, not a message, and a canonical one — which makes it a natural fit for databases and CRDTs alike. One wire format. Twelve byte-identical implementations. Zero schemas.

12 languages memcmp-orderable streamable canonical schema-free zero-dependency

The idea

A struple is a sequence of typed values packed into a byte buffer so that comparing the raw bytes gives you the right order:

std.mem.order(pack(a), pack(b))  ==  the semantic order of a and b

Every element starts with a one-byte type code, chosen so the type byte alone gives the cross-type order. Within a type, payloads are encoded to preserve order under memcmp: integers carry their width in the type code (negatives in excess form), floats use the IEEE-754 total-ordering transform, and variable-length values are 0x00-terminated so a shorter value sorts before a longer one that extends it. It's the FoundationDB tuple idea, rebuilt clean.

Two orders, one format bytes vs. value

The raw byte order keeps types from interleaving — an int always sorts before a float. When you want to compare by value instead, semanticOrder does it exactly, across every numeric representation.

The byte order always puts the int 5 before the float 2.5 (type code first); the value order sees 5 > 2.5. The real thing is exact even past 253, where a double can't represent the integer — try [5] vs [5.5] for a case where they agree.

Read it as it streams a stream of fields, not a message

A struple isn't a blob you parse all at once — it's a sequence of self-delimiting elements. Each one carries its own length in its type code and framing, so there's no length prefix to wait for.

48 7573657200 212a ▸ 06

A reader walks the stream one element at a time, so you can decode field 1, field 2, … as the bytes arrive — pull out the few fields you need and stop, or skip the rest without decoding it. That's the opposite of a random-access format like Cap'n Proto, which needs the whole message resident to follow its pointers.

And because each element leads with its most-significant bytes (type code first, then big-endian payload), a prefix of the stream already orders correctly — perfect for database key scans, range queries, and tailing a log.

Built for CRDTs deterministic + ordered = mergeable

Two properties make struple an unusually good substrate for conflict-free replicated data (CRDTs) and content-addressed systems:

Canonical bytes

The same value always encodes to exactly the same bytes — map keys sorted, floats canonicalized, no whitespace or ordering ambiguity — and identically across all twelve languages. So you can hash, compare, dedup, and merge by raw bytes, and two replicas independently produce byte-identical encodings. Most formats (JSON, Protobuf, MessagePack) have no canonical form at all.

Ordered keys

Because keys are memcmp-ordered, you can mint a key between any two existing keys — fractional indexing for sequence CRDTs (RGA / LSEQ / Fugue-style), ordered maps and sets that merge by range, and operation logs that sort deterministically — for tuples of any typed values, not just strings.

Deterministic encoding + a total order is exactly the substrate that replicated, mergeable, ordered data is built on.

Where it shines ordered + canonical + self-delimiting

Three properties — bytes that sort themselves, one canonical encoding per value, and a stream you can read incrementally — add up to a handful of things struple is unusually good at.

🔑 Database keys & indexes

Pack a composite key like [tenant, "user", id] and drop the bytes straight into RocksDB, LMDB, sled or FoundationDB. They sort in the right order with no custom comparator, so range scans, prefix queries and secondary indexes just work.

📨 Message passing & queues

Because all twelve languages emit identical bytes, a struple is a language-neutral message — no schema compiler, no codegen. It's how Substrate wires functional-style FFIs and queues between languages and cores: a Rust core enqueues a struple, a Python or Swift worker dequeues and decodes it. Self-delimiting, so you can parse it as it streams.

🧩 Content addressing & caching

One value → one canonical byte string → one hash. Use it as a cache key, a content address or a dedup fingerprint, and every language agrees on the identity. The same property makes it a clean CRDT substrate.

🌐 Polyglot data exchange

A wire format spanning the union of the Python and JavaScript data models — nulls, big integers, decimals, timestamps, UUIDs, nested maps and sets — round-tripping byte-identically across a dozen runtimes. One format for your whole stack, front to back.

How fast is it? same bytes, every language — pick a workload

One wire format means this doubles as a clean cross-language micro-benchmark: every port encodes and decodes the identical bytes (sha256-verified before timing). No favouritism — just whoever's fastest at each workload. Figures are single-threaded, millions of records per second; the work is embarrassingly parallel, so multiply by your cores.
